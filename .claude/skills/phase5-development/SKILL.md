# PHASE 5 — DEVELOPMENT SPRINTS

## SPRINT FLOW (1 user story per cycle)
1. Read story + acceptance criteria → architecture → design specs → CLIENT.txt
2. Plan files, data flow, errors, tests BEFORE coding. Note edge cases (PRD §8).
3. Ambiguous? log gap → ask PM. Never guess.
4. Implement + tests → run lint + types + tests + security scan
5. Verify against acceptance criteria
6. Update `contextlog.md` + `buglog.md`
7. `/compact` after each story · `/clear` after each sprint

Logs ARE memory across sessions — persist before compact/clear.

## RESEARCH
- Library / framework / API → **Context7** MCP first (current docs > training data)
- "Best" claims → search online, cite URL, state confidence

## DEBUGGING
1. Read full error → ask WHY, not how to silence
2. **Read PRD for the broken subsystem** — understand spec'd intent (retry policy, timeouts, fallback, error UX)
3. Trace data flow → fix root cause → log in `buglog.md` → add regression test
4. **Red flag:** any pattern not in spec (circuit breaker, exponential backoff, extra retries, extended timeouts) — challenge it; it likely contradicts the spec

Never: hack, wrap-and-ignore, change tests to match bugs, silence with `@ts-ignore`.

## PRINCIPLES — defaults, not laws
The rules below are defaults. Each has legitimate exceptions; document them in `contextlog.md` when you break one.

- **DRY** — default. Exception: when forcing DRY would create coupling, leak abstractions across boundaries, or generate technical debt worse than the duplication. AHA: duplicate once before abstracting; extract only when the pattern is stable.
- **KISS** — default. Exception: essentially-complex domains (concurrency, security, finance, schedulers) can't be made trivially simple without losing correctness. Eliminate accidental complexity; accept essential complexity.
- **YAGNI** — default. Build for this sprint, not hypothetical futures.
- **SOLID** — guidelines that reduce coupling. Skip when they only add boilerplate.
- **Boy Scout Rule** — leave cleaner, not rewritten. Improvements stay in ticket scope.
- **Done = tested locally** — never mark done without running it end-to-end on your machine.

## NAMING & FILES
Verbs for fns (`getUserById`) · `is/has/can` booleans · `UPPER_SNAKE` consts · `PascalCase` classes · `camelCase`/`snake_case` vars · no abbreviations.
One class/component per file · max 300 lines · group by feature · zero circular deps.

## TYPE SAFETY
`strict: true` (TS) / `mypy --strict`. Zero `any`, zero `@ts-ignore` without `buglog.md` entry. Runtime validation at boundaries with **Zod** (or Valibot). DTOs at boundaries — never expose raw DB models. Validate env vars at startup.

## FRONTEND
- **Mobile-first by default** — exception only when spec says desktop-only.
- **a11y is baseline** — semantic HTML + ARIA + keyboard nav. WCAG 2.1 AA.
- **Every async action** = loading + success + actionable error.
- **State** — never mutate. Local for UI-only; global store (Zustand) for shared.
- **Resilience** — optional chaining + defaults + Error Boundaries on data-driven UI.
- **Perf** — Lighthouse ≥90 · LCP <2s · lazy-load below-the-fold · design tokens, not inline styles.
- **API clients** — generated by **Orval** from OpenAPI; never hand-write fetch wrappers.
- **Scaffolding** — use `pnpm create vite <name> --template <preset>` (presets: react-ts, vue-ts, svelte-ts, solid-ts, lit-ts, preact-ts, qwik, vanilla-ts; `pnpm create vite-extra` for more). Don't burn tokens rewriting boilerplate Vite already generates — start from a template, then customize.

## BACKEND
- **REST** — plural noun resources (`/users`). OpenAPI spec is source of truth. Breaking change → `/v2/...`; never break an active endpoint.
- **Pagination** mandatory on lists — default 20, max 100. No fetch-all.
- **Edge validation** — type/length/format/range at the boundary, before business logic. Never trust the client.
- **Idempotency** on every mutating endpoint (key or natural). Prevents double-charges from retries/double-clicks.
- **Graceful shutdown** — handle `SIGTERM`: drain requests, close DB, exit.
- **Perf target** — <200ms p95.

## DATABASE
- Every table: `id` + `created_at` + `updated_at`, **all UTC**. Convert to local timezone on the frontend only.
- **Never `SELECT *`** — name columns explicitly.
- **Migrations** — reversible, non-locking. Never modify a deployed migration.
- Index FKs + WHERE columns · `EXPLAIN` new queries.
- Transactions for multi-step ops · constraints at DB level.
- **Soft deletes** preserve relational integrity. Hard delete only on GDPR/CCPA erasure.

## SECURITY (OWASP 2025)
**Identity — never roll your own.** Use **Keycloak**, **Auth0**, **Clerk**, or **NextAuth**.

**Every endpoint:** AuthN (logged in?) + AuthZ (does THIS user own THIS resource? — RBAC/ABAC).

- Parameterized queries only · escape output by context · CSRF on
- HTTPS only · CORS restricted (no wildcards) · rate limit · timeout external calls · fail CLOSED
- Passwords: **bcrypt** / **argon2**
- Secrets in env or secret manager — never committed, never in code
- Generic errors to users, details to logs (no stacks/paths/versions exposed)
- Lock dep versions · audit before adding packages
- **Least privilege** for services, DB users, humans

## ERROR HANDLING
- Layered: Route (HTTP status) → Service (typed exceptions) → Data (catch & wrap)
- Response shape: `{ error: { code, message, status, requestId } }`
- Codes: `400` bad input · `401` unauthenticated · `403` forbidden · `404` not found · `409` conflict · `422` validation · `500` server. Never `200` with `{ success: false }`.
- Handle: nulls, wrong types, duplicates, concurrent edits, expired sessions, network mid-action failures, empty/max states.

## TESTING
Pyramid: many unit (≥80%) → moderate integration → few e2e (critical paths).
Tools: **Vitest** (unit) · **Supertest** (API) · **Playwright** (e2e).
- Arrange–Act–Assert · one behavior per test · independent · descriptive names
- **Mock at boundaries only** (DB, external APIs) — never internal modules
- **Test negative paths** — malformed input, timeouts, empty responses, unauthorized access
- **Flaky = failing** — quarantine + fix immediately; never `--force` past
- TDD for complex logic; test-after allowed for exploratory UI (still required)

## OBSERVABILITY
- **No PII in logs** — no passwords/tokens/emails/cards/SSNs. Legal.
- **Centralized errors** — **Sentry** for unhandled exceptions
- Structured logs with `requestId`
- **Actionable alerts only** — page humans only when human action is required
- Log security events explicitly

## GIT & DELIVERY
- Branches: `feature/E001-US001-desc` · `fix/BUG-042-desc` · `hotfix/SEC-003-desc`
- Conventional commits: `<type>(<scope>): <desc>` — feat/fix/security/refactor/test/docs/chore/perf. Explain WHY.
- PRs: small + focused · CI must pass · review required · delete branch after merge · **never push to main**
- **pnpm always** — commit `pnpm-lock.yaml`
- **Immutable artifacts** — build Docker image once, promote staging → prod
- **Feature flags** — decouple deploy from release
- **Zero-downtime** — schema changes backward-compatible with running code

## CODE REVIEW CHECKLIST
Acceptance criteria · edge cases · AuthN+AuthZ · no secrets/PII · no N+1 · happy + negative tests · pagination on lists · logs updated · no `any`/`@ts-ignore`.
