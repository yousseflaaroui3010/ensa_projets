## Project Scaffold & Architecture — 2026-05-05

**What:** Set up the full project structure: configuration file, dependencies list, and all source code modules (data pipeline, model, inference engine, explainability, and web app).

**Why:** A clean separation of concerns (data / model / inference / explainability / UI in different files) means each part can be changed or tested independently. All numbers like thresholds and learning rates are stored in one `config.yaml` file instead of being scattered across the code — this makes tuning much easier.

**Result:** Full codebase scaffolded and ready for training. No model weights yet — those are generated by running `notebooks/02_training.ipynb`.

**Acronyms used:**
- **CNN** (Convolutional Neural Network): a type of AI model designed to process images by detecting patterns like edges, shapes, and textures.
- **EfficientNetB0**: a specific CNN architecture known for being small and fast. The "B0" means it is the smallest size in the EfficientNet family.
- **OOD** (Out-of-Distribution): an image that is completely different from what the model was trained on. Example: uploading a photo of a dog to a brain MRI classifier.
- **Grad-CAM** (Gradient-weighted Class Activation Mapping): a technique that produces a visual heatmap showing which parts of an image the AI focused on to make its decision.
- **Transfer Learning**: reusing a model already trained on millions of general images (ImageNet) and adapting it to a new specific task (brain tumor detection) — saves weeks of compute time.
- **Streamlit**: a Python library for building simple web interfaces without writing HTML or JavaScript.
